Introduction

A key feature of programming is its ability to automate repetitive tasks. One way to implement it is through for-loops, which allows the same set of instructions to be applied repeatedly, but without duplicating code. [note] Julia also provides while-loops, but their use is less pervasive and so we won't cover it.

In certain languages (e.g., Matlab, Python, and R), for-loops can negatively impact performance. These languages recommend instead using vectorized code, thus operating with the whole vector rather than its individual elements. Such a recommendation has no bearing on Julia, where for-loops are as fast as vectorized code. In fact, speeding up operations in Julia commonly rely on for-loops, as they're easy to optimize. [note] For instance, for-loops are easy to parallelize. This allows the computer to simultaneously use all its processor cores, which can greatly enhance performance. However, parallelized code also introduces two key challenges, explaining why they don't tend to be the default option. First, it exposes the code to unsafe operations. An example of this is the so-called race conditions. This occurs when the order in which the computer executes the operations matter, possibly corrupting output silently. The second issue is that parallelized code only entails performance gains for computationally intensive tasks, and actually results in slower code for small tasks.

For-Loops Syntax

For-loops run the same code block on each element of a list. To illustrate its application, consider println(a). This function evaluates a and displays its output in the REPL. In case a is a String variable, println(a) simply displays the word stored in a.

Code 

for x in ["hello","beautiful","world"]
    println(x)
end

Output in REPL
"hello" "beautiful" "world"

The syntax of for-loops requires the keyword for at the beginning and the keyword end at the end. Furthermore, for-loops have three components:

  • A code block to be executed: represented by println(x) in the example.

  • A list of elements: this is ["hello","beautiful","world"] in the example, and represents the elements over which we'll apply the code block. The list can consist of elements having any data type (e.g., strings, numbers, and even functions). The only restriction is that it has to be iterable object, defined as a container allowing the user to access each element individually. Examples of iterable objects are vectors (as in the example), but they're not the only option. One important iterable object that we'll explore is ranges, which specify vector indices.

  • An iteration variable: this corresponds to x in the example. It serves as a label that references each element in the list, with no significance outside the for-loop. As we expand upon below, it's a local variable.


Remark
As we discussed here, the keyword in is a binary operator, explaining the syntax x <operator> list. In fact, in can be replaced by âˆˆ, which is a binary operator having the same functionality, or even =. [note] Recall that âˆˆ can be written through Tab Completion using the command \in

The consequence is that the following are all equivalent.

    println(x)
end

    println(x)
end

end

Furthermore, we can use any term to describe the iteration variable.

An Alternative Name For The Iteration Variable 

for word in ["hello","beautiful","world"] 
    println(word)
end

Output in REPL
"hello" "beautiful" "world"

Iterating over Indices

Remark
At this stage of the website, we're still introducing the fundamental concepts of Julia, explaining why our examples make simplifications. In the particular case of for-loops, we'll be writing the for-loops in the global scope. However, you should always wrap for-loops in functions in real applications.

Structuring programs around multiple functions is a general best practice, but additionally entails substantial performance gains in Julia. This is especially true for for-loops, where executing them outside a function becomes a performance killer for medium to large tasks.

The example considered above was using ["hello", "beautiful", "world"] as the list to iterate over. In this list, the user must manually specify each element, one by one. We now explore various approaches for specifying a sequence of values to create an iterable object, which is crucial for iterating over vector indices.

Ranges

One of the most general techniques to create a sequence of values is through ranges. Recall that the syntax for defining a range is <begin>:<steps>:<end>, where <begin> and <end> respectively refer to the first and last index. The term <steps> refers to the increments between two consecutive values, with negative values reversing the range order. If you write <begin>:<end> and thus omit <steps>, Julia assumes steps equal to one.

for i in 1:2:5
    println(i)
end

Output in REPL
1 3 5

for i in 3:-1:1
    println(i)
end

Output in REPL
3 2 1

Remark
Ranges define a sequence of numbers, and so can be used for other purposes in addition to iterations in a for-loop.

For instance, ranges are helpful to define vectors, by combining them with the function collect.

Code 

x = collect(4:6)

Output in REPL
julia> x
3-element Vector{Int64}: 4 5 6

Iterating over All Indices of an Array

To iterate over all the indices of an array x, Julia provides the built-in function eachindex(x). This method is recommended by Julia's developers to iterate over all indices, as has been optimized to return a fast iterator for each type of array (e.g., vectors, matrices, and other types of arrays we haven't covered yet).

It's also possible to use ranges with the same purpose. However, unlike what occurs when ranges are employed for defining slices, the use of ranges for creating an iterable object doesn't allow for the syntax begin:end. Instead, we have to employ 1:length(x), where length(x) indicates the number of elements in x. 

x = [4, 6, 8]

for i in eachindex(x)
    println(x[i])
end

Output in REPL
4 6 8

x = [4, 6, 8]

for i in 1:length(x)
    println(x[i])
end

Output in REPL
4 6 8

Remark
Note we could also iterate over all indices of a vector through 1:length(x), where length(x) indicates the number of elements in x. Indeed, you'll see that there are multiple approaches to doing this. For instance, the function LinearIndices(x) returns a vector with all the indices of an array. Another possibility is firstindex(x):lastindex(x), which makes it possible to specify a range from the first to the last index of x

The multiplicity of methods similar in appearance is to cover "unconventional" indices arising from the package OffsetArrays.jl. This package makes Julia's arrays have 0 as the first index, as is common in other languages. When this occurs, 1 differs from firstindex(x) and length(x) is different from lastindex(x).

Unless you create a package that other users will be using, you don't have to worry about itâ€”these functions can be used interchangeably. This is shown in the following examples.

x = [4, 6, 8]

for i in eachindex(x)
    println(x[i])
end

Output in REPL
4 6 8

x = [4, 6, 8]

for i in 1:length(x)
    println(x[i])
end

Output in REPL
4 6 8

x = [4, 6, 8]

for i in LinearIndices(x) 
    println(x[i])
end

Output in REPL
4 6 8

x = [4, 6, 8]

for i in firstindex(x):lastindex(x)
    println(x[i])
end

Output in REPL
4 6 8

Even if all the functions could be applied, it's best to directly use eachindex. This function is optimized and constitutes a valid iterator for any vector, irrespective of how it's indexed.

Rules for Variable Scope in For-Loops

For-loops create a new scope for variables, analogous to what occurs with functions. In fact, the scoping rules for functions and for-loops are quite similarâ€” the main difference is that for-loops can redefine the value of a global variable, while functions can't.

Warning!
The rules of variable scope that we present hold in all circumstances, except for one specific case. This case is rare and mainly arises from following bad practices, explaining why we won't cover it.

The issue arises specifically when i) the for-loop is not inside a function, ii) a local variable shares its name with a global variable, and iii) the script is run non-interactively (i.e., by executing the function include). [note] Recall that there are two methods to execute a script, as we saw here. The first method consists in working interactively with Julia, which covers running commands in the REPL's prompt julia>and the execution of a script through a code editor. The second method is by executing a file that stores a script through the function include.

Unless the three conditions hold simultaneously, you don't have to worry about it. And even if this occurs, Julia will display a warning in the REPL indicating that there's a problem with your code.

We describe the variable scope by referring to a variable x. The rules are as follows:

  • the variable of iteration x is always local, regardless of whether there's a variable x defined outside the for-loop.

  • if there is no variable named x outside the for-loop, then x is a new local variable. This variable won't be accessible outside the for-loop.

  • if there is a variable named x outside the for-loop, then x refers to this variable.

The following examples illustrate the first two rules, which refer to local variables exclusively. The second example in particular reflects a common mistake of beginners: running a for-loop with a local variable, and then trying to access it outside the for-loop. 

x = 2

for x in ["hello"]          # this 'x' is local, not related to 'x = 2'
    println(x)
end

Output in REPL
"hello"

#no `x` outside the for-loop

for word in ["hello"]
    x = word                # `x` is local to the for-loop, not available outside it
end

Output in REPL
julia> x
ERROR: UndefVarError: x not defined

Next, we show the consequences of variable scope for global variables. They're summarized by the last rule we mentioned above.

x = [2, 4, 6]

for i in eachindex(x)
    x[i] * 10        # it refers to the `x` outside of the for-loop
end

Output in REPL
julia> x
3-element Vector{Int64}: 20 40 60

x = [2, 4, 6]

for word in ["hello"]
    x = word                        # it reassigns the `x` defined outside the for-loop
end

Output in REPL
julia> x
"hello"

Array Comprehensions

The previous section showed that we can define an array by following a two-step procedure: we initialize an array, and then set its values through a for-loop. The fact that we need to perform two operations is inconvenient and verbose. An alternative is given by array comprehensions, which makes it possible to create arrays through for-loops, but with a more concise syntax:[<expression> for... if...], where <expression> can be an operation or a function.

To illustrate its use, consider a vector x. The goal is to create a vector y whos elements are x's elements squared. [note] This example only aims at showing the syntax of comprehensionsâ€”y could be simply created by y = x .* x or y = x .^ 2. The following examples show two approaches to creating y through a comprehension. 

x = [1,2,3]


y = [a^2 for a in x]                  # or y = [x[i]^2 for i in eachindex(x)]

Output in REPL
julia> y
3-element Vector{Int64}: 1 4 9

x = [1,2,3]

foo(a) = a^2
y      = [foo(a) for a in x]          # or y = [foo(x[i]) for i in eachindex(x)]

Output in REPL
julia> y
3-element Vector{Int64}: 1 4 9

You can also add conditions to comprehensions, which are placed at the end of the expression.

Comprehension with Condition 

x = [i for i in 1:4 if i â‰¤ 3]

Output in REPL
julia> x
3-element Vector{Int64}: 1 2 3
Remark
Array comprehensions can also be used to create matrices. In terms of syntax, this requires using a comma and specifying the second dimension.

Comprehension for Matrices 

y = [i * j for i in 1:2, j in 1:2]

Output in REPL
julia> y
2Ã—2 Matrix{Int64}: 1 2 2 4

Multiple Iterations

Thus far, we've considered for-loops that iterate over single values. We now extend the applications of for-loops to iterate over multiple values. We study in particular two scenarios: the simultaneous iteration over two lists, and over indices and values of a vector.

Iterating over Two Lists

There are two possibilities to iterate simultaneously over two lists x and y, depending on the combination of elements aimed. First, we could iterate over all the possible combinations of their elements. This requires using the function Iterators.product(x,y), where the package Iterators is "imported" by default in each Julia session.

Alternatively, we could iterate over all the ordered pairs of x and y, meaning that each iteration considers the i-th element of x and y. This operation is implemented through the function zip(x,y).

The following examples show their use. 

list1 = [1, 2]
list2 = [3, 4]

for (a,b) in Iterators.product(list1,list2) #it takes all possible combinations
    println([a,b])
end

Output in REPL
[1,3] [2,3] [1,4] [2,4]

list1 = [1, 2]
list2 = [3, 4]

for (a,b) in zip(list1,list2)               #it takes pairs of elements with the same index
    println([a,b])
end

Output in REPL
[1,3] [2,4]

You can also iterate over multiple values using array comprehensions. The syntax for all the possible combinations of two lists requires repeating for twice, while zip is used for taking pairs.

x = [i * j for i in 1:2 for j in 1:2]

Output in REPL
julia> x
4-element Vector{Int64}: 1 2 2 4

x = [i * j for (i,j) in zip(1:2, 1:2)]

Output in REPL
julia> x
2-element Vector{Int64}: 1 4

Simultaneously Iterating over Indices and Values

Given a vector x, we can also iterate over each pair of index-value. This is implemented through the function enumerate.

x = ["hello", "world"]

for (index,value) in enumerate(x)
    println("$index $value")
end

Output in REPL
"1 hello" "2 world"

x = [10, 20]


y = [index * value for (index,value) in enumerate(x)]

Output in REPL
julia> y
2-element Vector{Int64}: 10 40

Iterating over Functions

Functions in Julia are first-class objects. This means that functions can be manipulated in a similar way as other primitive objects, such as strings and numbers. In particular, we can define a vector of functions and iterate over them. The next example makes use of this feature, by characterizing a vector x through several descriptive statistics.

Code 

using Statistics    # to get access to the function `mean` and `median`

x              = [10, 50, 100]
list_functions = [maximum, minimum, mean, median]

descriptive(vector,list) = [foo(vector) for foo in list]

Output in REPL
julia> descriptive(x, list_functions)
4-element Vector{Real}: 100 10 53.333333333333336 50.0
