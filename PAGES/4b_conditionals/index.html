Introduction

Conditional statements are contingent executions of code blocks, where each code block is referred to as a branch. Based on the number of branches, there are three types of conditional statements:

  • if-then statements, which consist of a single branch. They run a specific operation only if a condition is met, with no operation performed otherwise.

  • if-else statements, which consist of two branches. They run a specific operation if a condition is met, and another if the condition isn't satisfied.

  • if-else-if statements, which consist of three or more branches. They comprise a series of conditions, with each branch executing a different code block.

The presentation builds heavily on the logical operators introduced in the previous section. If you haven't read it, I highly recommend doing so before continuing.

If-Then Statements

The primary function of if-then statements is to execute an operation when a condition is met, but none if it isn't satisfied. There are three distinct ways to construct if-then statements: one is based on the if keyword, while the other two employ the logical operators && and ||.

The syntax of each form can be observed below. The examples rely on the println function, which outputs the provided text to the REPL.

x = 5

if x > 0
    println("x is positive")
end

Output in REPL
"x is positive" 

x = 5

(x > 0) && (println("x is positive"))

Output in REPL
"x is positive" 

x = 5

(x â‰¤ 0) || (println("x is positive"))

Output in REPL
"x is positive"

If-then statements imply that no action would've been taken if x = -1â€”it's only when x > 0 that println is executed. The if approach is somewhat verbose for simple conditional statements. However, we'll see that it also constitutes the most versatile option. It's usually reserved for more complex conditional statements, where multiple branches are included.

In contrast, && and || are more concise, helping us keep the code streamlined. The operator && specifically executes an operation if the condition considered is true, whereas || runs an operation when the condition is not satisfied. In fact, || is equivalent to && with its condition negated. This can be appreciated in the example above, where we obtain the same results by inverting the condition x â‰¤ 0 through !(x > 0).

Remark
Note that these operators act as if-then statements because, although the first operand is a condition, the second operand is an operation. This is in contrast to the use of && and || for conditions, where all operands must be Bool values.

If-then statements using || are commonly applied to alert an issue with the operation. This requires combining || with the function error, which immediately interrupts the script's execution and displays the message given by its argument. For instance, if you define a function foo(x) that only accepts a non-negative value for x (e.g., the logarithmic function), you could include x > 0 || error("x must be positive") as the function's first line. Then, running foo(x) will stop its execution if x is not positive, with the message "x must be positive" displayed in the REPL.

If-Else Statements

If-else statements execute an operation when a condition is met and execute another one when it isn't. These statements can be represented in two forms.

The first approach is the most flexible, and extends the use of if through the keyword else. The alternative is based on the so-called ternary operator, which requires the keywords ? and :. Specifically, it's given by <condition> ? <operation if true> : <operation if false>. It's the only operator that takes three arguments, explaining why it's called the ternary operator.

We illustrate the syntax of each approach below.

x = 5

if x > 0
    println("x is positive")
else
    println("x is not positive")
end

Output in REPL
"x is positive" 

x = 5

x > 0 ? println("x is positive") : println("x is not positive")

Output in REPL
"x is positive"

The function ifelse provides an alternative for defining if-else expressions. The function takes three arguments: a condition, an expression to be executed if true, and another expression to execute if false.

Importantly, the function ifelse doesn't act as a short-circuit operator, meaning that all the statements are evaluated regardless of whether the condition is true.

The use of functions allows for broadcasting operations. This is especially helpful for defining vectors, whose elements adopt different values depending on a condition. This is demonstrated below.

x = [4, 2, -6]

are_elements_positive = ifelse.(x .> 0, true, false)

Output in REPL
julia>
are_elements_positive
3-element BitVector: 1 1 0

x = [4, 2, -6]

x_absolute_value = ifelse.(x .â‰¥ 0, x, -x)

Output in REPL
julia>
x_absolute_value
3-element Vector{Int64}: 4 2 6

Remark
Notice that broadcasting ifelse requires broadcasting both ifelse and the condition. The first example, for case, would throw an error if we only broadcast ifelse through ifelse.(x>0, true, false). This occurs because x > 0 would be checking whether the entire vector is positive, which is an undefined operation in Julia.

If-Else-If Statements

So far, weâ€™ve analyzed conditional statements that consist of at most two operations: one if the condition is met, and another if it isnâ€™t. However, this structure has two limitations when dealing with multiple alternatives. Firstly, it requires us to define an operation for each possible scenario, as the operation executed when the condition isnâ€™t met acts as a catch-all term. Secondly, it doesnâ€™t allow the user to handle more than two options.

Both restrictions can be circumvented through adding the keyword elseif to the approach with if and else. We present two examples and then elaborate on each. 

x = -10

if x > 0
    println("x is positive")
elseif x == 0
    println("x is zero")
end

Output in REPL 

x = -10

if x > 0
    println("x is positive")
elseif x == 0
    println("x is zero")
else
    println("x is negative")
end

Output in REPL
"x is negative"

The first example demonstrates that elseif eliminates the need to specify what occurs in each scenario. Specifically, the conditional statement performs an action if x is positive, another action if x is zero, but does nothing otherwise. Likewise, the second example reveals the flexibility of this approach, enabling the user to combine the if, else, and elseif keywords.
