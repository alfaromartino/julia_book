Introduction

This section introduces the concept of variables, types, and operators. We'll also cover the syntax of operators, which varies depending on the number of inputs they accept.

To ensure a smooth learning experience, I've minimized the reliance on objects we haven't covered yet. The only new one introduced is vectors, whose syntax requires enclosing elements in brackets (e.g., [1,2,3]).

Creating Variables

Variables are created using the so-called assignment operator =. For instance, x = 3 binds x to an object holding the value 3.

A variable's name can be defined using Unicode characters, allowing for a wide range of possibilities. This includes Greek letters, Chinese characters, symbols, and even emoticons. [note] There are plenty of websites offering lists of Unicode characters(e.g. this website). They allow you to insert a specific Unicode character by copying and pasting it. You can alternatively use Tab completion, given the commands provided on this list. Variable names also accept underscores _, which are helpful to separate names with multiple words (e.g., intermediate_result). Finally, names are case-sensitive, so that bar and Bar are two distinct variables.

Code 

a = 2
A = 2               # variable `A` is different from `a`

new_value = 2       # underscores allowed

β = 2               # Greek letters allowed

中國 = 2            # Chinese characters allowed   

x̄  = 2              # decorations allowed
x₁ = 2               

🐒 = 2             # emoticons allowed

×

Names not Allowed

Variables can start with a letter or _, but never with a number. Moreover, symbols are not generally allowed.

Code 

_x1 = 2 #this is allowed

1x  = 2 #this throws an error
x^  = 2 #this throws an error
%x  = 2 #this throws an error

A variable’s value can be updated. This means that you can assign a new value to a variable by using the variable itself.

Code 

x  = 2

x  = x + 3   # 'x' now equals 5

x += 3       #shorthand notation for 'x = x + 3'

Warning!
Unlike other programming languages, Julia doesn't let you delete variables. Once a variable is created, it remains in memory until the program terminates. If a variable is taking up too much memory, you can free up space by assigning a lighter object to it.

What are Variables?

Formally, variables are labels for objects, and hence act as mere identifiers. In turn, objects are storage locations that hold values. [note] Storage locations are abstract, meaning that they don't necessarily refer to a physical memory address in RAM. The actual memory location of the object may change over time and not even reside in RAM. Internally, programming languages handle object storage depending on various factors such as available memory, cache performance, and garbage collection.

Objects can be characterized by their values, which have associated a type that identifies the nature of the object. For example, x = "Hello" is a variable pointing to an object whose value has a type named String. [note] To avoid verbosity, throughout the website we'll say that "a variable has a specific type". Strictly speaking, this is inaccurate —it's actually a variable's value that has a specific type, not the variable itself. While this distinction may seem unimportant, it'll become relevant when discussing more advanced topics such as type stability.

Variables


Given the importance of types, we discuss them in more detail in the next section.

Types

The objects referenced by a variable can hold either a single element or a collection of elements. Collections are also known as containers, and we refer to them interchangeably throughout the website. Below, we provide a brief overview of single-element objects and collections.

Objects


The diagram above describes informally the nature of an object's value. More precisely, each object holds a value that is described in Julia through a type. The most common types for numeric single-element variables are Int64 for integers, Float64 for decimal numbers, and Bool for binary values. Likewise, the type Char represent characters. It serves as the building block for the type String, which are a collection of characters used to represent text.

When it comes to collections, their types usually describe both the nature of the object (e.g., a vector, a matrix) and the types of its elements (e.g., integer, floats). For example, [2,4,6] has type Vector{Int64}, which means that the object is a Vector and its elements have type Int64. [note] Given a variable x, you can identify its type by executing typeof(x), and its element's type by eltype(x). For instance, if x = [2,4,6], the output of typeof(x) is Vector{Int64} and the output of eltype(x) is Int64 The arguments enclosed within {} are actually the type's parameters, and their content can vary when we consider other types.

Remark - Restricting a Variable's Type
A variable's type can be restricted by using type annotations, which are implemented through the operator ::. For example, declaring x::String restricts x to store strings exclusively, with any attempt to assign a value of a different type resulting in an error.
Remark - Notation
Developers of Julia and their packages denote types by using capitalized words, such as Float64 or String. Note that this is merely a convention and not a requirement for defining types.

Type Hierarchy: Concrete Types and Abstract Types

Types in Julia follow a hierarchy, meaning that we can define supersets and subsets of types. For instance, the type Any is a superset comprising all the possible types in Julia (e.g., String, Int64, Float64). In the context of programming, supersets of types are also referred as supertypes.

These concepts give rise to the notions of concrete types and abstract types. Specifically, abstract types refer to a set of types that serve as a parent to other types. Conversely, concrete types refer to types that don't have subtypes, thereby acting as primitive objects. For instance, Any is an abstract type, whereas Float64 is a concrete type.

Abstract types provide great flexibility when we write code. For example, they allow the user to define a variable as a number, without specifying whether it's an integer or a float. This is accomplished by the built-in type Number, which contains all the possible numeric types (e.g., Float64, Int64, and others).

Users can build their own supertypes by using the type Union{<type1>, <type2>}, which in fact allows for multiple arguments. Such representation arises naturally in data analysis, where empty entries are represented by the type Missing. Thus, if, for instance, we load a dataset having a column with integers and empty entries, the column will be stored with type Vector{Union{Missing,Int64}}.

Operators

Operators are symbols that perform operations on objects. For instance, the expression x + y performs the addition of x and y, and returns their sum as its output. The symbol + in this expression is called an operator. Likewise, x and y are its operands, which represent the operator's inputs to perform the calculation.

Operators follow specific syntax rules when they're classified by their number of operands. Next, we present this syntax through several examples. Just focus on how operators are written, even if you don't fully understand what they do in the examples. Identifying the syntax of operators will be useful when we discuss vectorized operations.

  • Unary operators: They use one operand, with the operator written to the left of the operand. [note] Operators to the left of the operator are known as prefix operators. Conversely, operators written to the right of the operand are known as postfix operators, and Julia has a few of them (e.g., ' to transpose a vector or matrix x is written by x'). We won't use postfix operators throughout this website.

    The syntax is <operator>x, such as √x or -x.

  • Binary operators: They use two operands, and the operator is written between the operands. [note] Operators with this syntax are called infix operators. The syntax is x <operator> y, such as x + y or x^y (for xyx^yxy).

  • Ternary operators: They use three operands and are rare. That's why x ? y : z is usually referred to as the ternary operator. This executes a conditional evaluation, with x representing the condition. It's such that y is returned if x is true, and z is returned otherwise.
